# 参与贡献

感谢你愿意帮助“妆容参照”变得更有用。这里欢迎产品反馈、无障碍与移动端改进、隐私与安全建议、文档修正、测试和代码贡献。

## 先选择正确入口

- 发现可以稳定复现的问题：提交 Bug Issue。
- 想讨论新功能或使用场景：先到 Discussions 说明问题，而不是直接给出一套大改方案。
- 已经明确的小改动：可以直接提交 Pull Request。
- 博主本人申请加入匹配库：使用正式服务中的“博主申请”，不要通过 GitHub 提交。
- 安全漏洞：按照 [安全政策](SECURITY.md) 私下报告。

## 隐私红线

GitHub 是公开协作空间。请勿在 Issue、Discussion、Pull Request、测试夹具或截图中提交：

- 真人正脸照、面部特征或匹配结果；
- 邮箱、手机号、私信记录或其他联系方式；
- 博主申请表、授权证明和审核记录；
- 生产数据库导出、访问令牌、密钥或服务日志中的个人信息。

复现问题时请使用虚构数据、合成图片或已彻底去标识化的最小样例。误传敏感信息后，请立即删除内容并通过正式部署页面公布的隐私联系邮箱通知维护者。

## 开始开发

环境要求：Node.js 20.19 及以上的 20.x 版本，或 Node.js 22.12 及以上版本。

```bash
npm install
npm run dev
```

提交前运行：

```bash
npm test
npm run build
```

普通用户的匹配照片和面部比例必须继续在浏览器本地处理。任何准备上传这些数据、扩大博主资料用途或改变授权范围的改动，都应先在 Issue 或 Discussion 中说明隐私影响。

## Pull Request 要求

- 一次 PR 解决一个明确问题。
- 描述用户场景、行为变化和验证方式。
- 用户界面改动附上不含真人或敏感信息的截图。
- 新行为有相应测试；纯文档修改可说明无需测试。
- 不顺手重构无关代码，不提交构建产物、密钥或生产数据。
- 确认 `npm test` 与 `npm run build` 通过。
