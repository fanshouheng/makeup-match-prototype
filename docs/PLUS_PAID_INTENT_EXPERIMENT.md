# LOOK AI Plus 付费意向实验

状态：已上线

开始日期：2026-07-24

## 目标

验证完成女生妆容分析的用户，是否愿意为以下完整闭环付费：

1. 基于浏览器本地结构数据生成详细面部特征报告；
2. 按一个或多个场景和妆造方向生成多套妆容参考方案；
3. 根据确认的方案联网查找相关美妆博主公开线索。

本轮只验证购买意向。不会收费，不会调用 Plus AI，不会收集联系方式或支付资料。

## 用户与入口

- 目标用户：完成女生妆容分析并看到结果的用户。
- 展示位置：免费匹配结果之后、现有 AI 联网推荐之前。
- 免费功能保持不变，不隐藏现有三位匹配博主、匹配原因、主页和教程链接。
- 男生形象参考暂不展示 Plus 实验。

## 产品承诺

### 1. 面部特征报告

- 使用现有九项面部结构数据；
- 使用中性、描述性语言，不做颜值评分、身份识别、医学结论或敏感属性推断；
- 正式开发时优先在浏览器本地生成，低置信度信息明确标注不确定。

### 2. 场景妆容方案

- 用户可以从八个常用场景中选择，也可以用最多 80 字直接描述自己的场景；
- 预设与自定义场景合计一至三个，自定义文字只保留在当前页面；
- 用户可以从六个妆造方向中选择一个，也可以选择“让 AI 建议”；
- 正式开发时每个场景生成日常稳妥版和重点表达版两套方案；
- 方案属于妆容参考，不承诺专业效果。

### 3. 按方案发现博主

- 正式开发时只按方案、场景和内容关键词搜索；
- 不下载或分析候选博主照片，不声称候选人与用户长相相似；
- 搜索结果与已授权创作者库分开展示，并标注为未完成主页与授权核验的公开线索。

## 实验流程

1. 结果页展示 Plus 价值和随机价格；
2. 用户点击“配置我的 Plus”；
3. 用户在本地选择或填写场景，并选择妆造方向；
4. 用户完成配置后，先看到一份明确标注的固定示例交付，包含面部结构报告、同一场景的两套六区域妆容方案和博主搜索简报；
5. 用户看完示例后选择“愿意为完整版付费”“内容有用，但价格高”或“暂时不需要”；
6. 页面始终说明本次不会扣款，也不会收集支付信息。

预设场景、自定义文字与妆造方向只用于当前页面交互，不发送到统计系统。示例固定以“毕业典礼 + 清透自然”为例，不基于用户照片、不调用 Plus AI，也不伪装成用户的实际结果。

## 价格分组

同一匿名标签页固定进入一个分组：

| 分组 | 展示价格 |
| --- | ---: |
| `price_9_9` | 9.9 元 / 次 |
| `price_19_9` | 19.9 元 / 次 |
| `price_29_9` | 29.9 元 / 次 |

分组由现有匿名会话 UUID 确定。刷新页面不会更换价格，不使用设备标识。

## 匿名事件

| 事件 | 含义 |
| --- | --- |
| `plus_offer_viewed` | Plus 方案进入可见结果页 |
| `plus_offer_opened` | 用户开始配置 Plus |
| `plus_offer_configured` | 用户选完至少一个场景和一个妆造方向，并打开示例交付 |
| `plus_intent_yes` | 用户表示愿意按当前价格购买 |
| `plus_intent_price_high` | 用户认为当前价格偏高 |
| `plus_intent_not_needed` | 用户暂时不需要 |

Supabase 只保存随机会话 UUID、固定事件名、价格分组和时间。不得保存照片、面部比例、匹配分数、场景选择、妆造方向、创作者信息、AI 结果、联系方式或支付资料。

## 判断规则

每个价格分组至少获得 300 次 `plus_offer_viewed` 后判断：

- 购买意向率低于 1%：停止开发真实 Plus；
- 1% 至 3%：保留方向，重新检查价值表达和交付内容；
- 高于 3%：进入真实 Plus 最小版本；
- 高于 5%：准备真实支付和交付闭环。

购买意向率定义为：

`plus_intent_yes / plus_offer_viewed`

同时观察配置完成率、价格偏高率和免费链路指标。意向点击不能作为真实收入对外宣传。

## 护栏与停止条件

- 页面必须在任何意向选择前说明“不会扣款”；
- 示例必须明确写明“不基于你的照片”，不能伪装成用户的 AI 生成结果；
- 不模拟支付表单，不要求登录，不收集联系方式；
- 不让付费改变匹配排名；
- 不因实验发送照片或面部数据；
- 免费结果交互明显受阻、投诉增加或埋点越界时立即下线实验；
- 正式开发前必须把化妆 Skill 转换为版本化规则和严格 JSON 输出协议。
- 妆造生成和豆包搜索的提示词边界见 `docs/PLUS_MAKEUP_RECOMMENDATION_PROMPT.md`。

## 下一阶段

只有实验达到进入线后，才实施真实 Plus：

1. 本地面部报告；
2. 用户自填信息与单独 AI 同意；
3. 妆容方案生成；
4. 独立的联网创作者搜索；
5. 输出校验、失败恢复和真实支付。
